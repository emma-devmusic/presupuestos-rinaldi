Presupuesto de Proyecto

Sistema de Stock Local + Sitio Web Ecommerce Básico

1. Descripción General del Proyecto

El objetivo principal de este proyecto es desarrollar un sistema de stock que funcione de manera local (offline) y que cuente con la capacidad de sincronizarse con un sitio web de comercio electrónico básico.

El sistema de stock permitirá gestionar inventarios y actualizar el portal web bajo demanda o cuando exista conexión a internet disponible. Por su parte, el sitio web cumplirá la función de generar órdenes de compra, las cuales quedarán pendientes de confirmación por parte del vendedor (sin incluir pasarelas de pago en esta etapa).

2. Alcance del Proyecto

Sistema de Stock (Local)

  • Gestión de Productos: Altas, bajas y modificaciones de productos, con sus respectivas descripciones, precios y códigos de identificación.
  • Control de Inventario: Control de existencias, registro de entradas y salidas de productos.
  • Informes Básicos: Reportes de stock disponible, ventas y alertas de bajo stock.
  • Sincronización con la Web:
    • Actualización de stock en el sitio web cuando el sistema local tenga conexión a internet.
    • Posibilidad de forzar la sincronización de manera manual o programada.

Sitio Web (Ecommerce Básico)

  • Catálogo de Productos: Visualización de la lista de productos, detalles, precios y disponibilidad (según el stock sincronizado).
  • Carrito de Compras: El usuario podrá agregar productos al carrito y generar una orden de compra.
  • Generación de Órdenes: Cuando el cliente confirma su compra, se crea una orden que queda pendiente de confirmación por parte del vendedor.
  • Gestión de Órdenes: Panel de administración para visualizar, confirmar, editar o cancelar órdenes.
  • Diseño Responsivo: Adaptación del sitio a dispositivos móviles, tabletas y computadoras de escritorio.

Exclusiones (Fuera de Alcance)

  • Integración con Pasarelas de Pago: No se incluye la configuración de métodos de pago en línea.
  • Desarrollo de Aplicaciones Móviles Nativas: No se considera la creación de apps más allá de la compatibilidad responsiva.
  • Integraciones con Terceros: No se incluyen integraciones con otros sistemas o software externos.

3. Objetivos del Proyecto

  1. Optimizar la Gestión de Inventarios: Control preciso y actualizado de stock, con posibilidad de sincronizar datos con el sitio web.
  2. Centralizar la Información de Ventas: Un sistema único que consolide todas las órdenes generadas en línea y las ventas locales.
  3. Mejorar la Experiencia del Cliente: Plataforma web sencilla para navegar, seleccionar productos y solicitar una orden de compra.
  4. Facilitar la Toma de Decisiones: Reportes básicos que apoyen en la gestión comercial y análisis de ventas.

4. Plan de Trabajo y Metodología

  1. Fase de Requerimientos:
    • Reuniones iniciales para detallar necesidades y especificaciones.
    • Documentación de requerimientos funcionales y no funcionales.
  2. Fase de Diseño:
    • Diseño de la base de datos local (productos, stock, órdenes, etc.).
    • Diseño de la arquitectura de sincronización (método y frecuencia).
    • Bocetos y wireframes del sitio web (layout, navegación).
  3. Fase de Desarrollo:
    • Desarrollo del Sistema de Stock Local:
      • Estructura de la aplicación y módulos de gestión de productos, inventario y reportes.
      • Programación de los mecanismos de sincronización con la web.
    • Desarrollo del Sitio Web:
      • Implementación del front-end (HTML, CSS, JS o framework a definir).
      • Implementación del back-end (servidor, base de datos en la nube, lógica de negocios).
      • Creación del panel de administración para gestionar productos y órdenes.
  4. Fase de Pruebas e Integración:
    • Pruebas unitarias y de integración (aplicación local y sitio web).
    • Verificación de la sincronización de datos (offline y online).
    • Resolución de errores y optimizaciones.
  5. Fase de Implementación y Capacitación:
    • Configuración del entorno de producción (hosting, instalación del sistema local).
    • Capacitación a los usuarios encargados de operar el sistema.
  6. Fase de Mantenimiento y Soporte Inicial:
    • Mantenimiento correctivo y preventivo durante un período acordado.
    • Posibles mejoras y ajuste de funcionalidades según la retroalimentación.

5. Entregables

  • Documentación de Requerimientos: Descripción detallada de funciones y características del proyecto.
  • Sistema de Stock Local: Aplicación con gestión de productos, inventario y reportes.
  • Sitio Web Funcional: Portal de ecommerce básico con carrito y generación de órdenes de compra, además de un panel de administración.
  • Módulo de Sincronización: Integrado en ambos sistemas para mantener la información de stock actualizada.
  • Manual de Usuario y Capacitación: Guía de uso para el sistema local y el sitio web, con sesiones de capacitación incluidas.
